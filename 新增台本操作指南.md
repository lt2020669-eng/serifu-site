# 新增台本 · 操作指南

> 目标：把一份新台本变成网站上现在这种格式，并上线。
> 全程不用自己写代码，交给 AI 助手（Cursor 的 Agent 模式）转换即可。

---

## 一图看懂流程

```
新台本文件  →  放进 inbox/  →  对 AI 说一句话转换  →  双击 start.bat 预览  →  git 推送上线
```

一份台本大约 5 分钟搞定。

---

## 准备：新台本长什么样都行

新台本可以是下面任意一种：

- Word 文档（`.docx`）
- Google 文档（导出成 `.docx` 或 `.md`）
- Markdown / 纯文本（`.md` / `.txt`）
- 网页文件（`.html`，就像最早那 5 份）

> 内容里日语汉字直接写就行，注音（ふりがな）交给 AI 加；已知读音也可以自己用「漢字(かんじ)」标一下帮它。

---

## 第 1 步：把文件放进 `inbox/`

把新台本文件复制到这个文件夹：

```
D:\食品\inbox\
```

文件名随意，例如 `inbox\新款煎饼台本.docx`。

---

## 第 2 步：对 AI 助手说一句话（转换）

在 **Cursor** 里打开 `D:\食品` 文件夹，切到 **Agent 模式**，把下面这句话复制发给它（把文件名换成你的）：

> 把 `inbox/新款煎饼台本.docx` 按 SCHEMA 转成标准台本，更新 index，归档源文件。

AI 会自动完成这些事（这套规则写在 `.cursor/skills/script-convert` 里）：

1. 按 `SCHEMA.md` 的格式生成 `content/scripts/{id}.json`
2. 更新总索引 `content/index.json`（新台本才会出现在网站列表里）
3. 给日语加上 ふりがな 注音
4. 把 `inbox/` 里的原文件移到 `archive/` 存档（不会删）
5. 在回复里列出 **「⚠ 需人工校对读音」** 清单

> 💡 你也可以顺便告诉它这份台本的信息，转得更准：行业（食品/服装…）、台本类型（商品资料卡 / 完整台本 / 介绍要点）、几个标签（tags）、商品完整名。

---

## 第 3 步：核对 AI 的产出

AI 转完后，**重点看它列出的「需人工校对读音」清单**——日语汉字有些读音会有多种，AI 会尽力标但可能出错。有拿不准的，直接让它改，例如：

> 「一日」在第 3 节应该读 ついたち，帮我改过来。

---

## 第 4 步：本地预览确认

**双击** `D:\食品\start.bat`，会自动打开浏览器。

- 输入验证码进入
- 到对应行业列表里找到新台本，点进去
- 检查：注音对不对、FAQ 全不全、图片有没有

看着没问题就进入下一步。（预览用完把那个黑色命令行窗口关掉即可。）

---

## 第 5 步：推送上线

在 `D:\食品` 目录打开终端（或让 Cursor 帮你跑），依次执行：

```powershell
git add -A
git commit -m "新增台本：新款煎饼"
git push
```

约 1 分钟后，线上就更新了：
**https://lt2020669-eng.github.io/serifu-site/**

---

## 如果是「自己从零写」一份台本

不是拿到现成台本、而是自己写，就先用模板保证结构统一：

1. 复制 `templates/script-input.md` 的内容到一个 Google 文档
2. **只填内容**，别改章节标题、别加颜色/样式
3. 导出为 `.md` 或 `.docx` 放进 `inbox/`
4. 之后同上：第 2 步 → 第 5 步

---

## 常见问题

**Q：网站列表里没看到新台本？**
A：多半是 `content/index.json` 没更新。让 AI「把这份台本补进 index.json」，或重跑第 2 步那句话。

**Q：一次有好几份台本？**
A：一份一份来，逐个放进 inbox、逐个说转换。别一句话让它转一批，容易出错。

**Q：图片怎么办？**
A：台本里的图片，AI 会存到 `site/assets/images/` 并在 JSON 里引用。你也可以自己把图片放进那个文件夹，把文件名告诉 AI。

**Q：改验证码？**
A：改 `site/assets/passcode.js` 里那一行数字，再 `git push`。详见 `README.md` 的「访问验证」。

**Q：id 是什么、会不会撞车？**
A：`id` 是每份台本的唯一英文代号（文件名也用它）。AI 会自动起、并避免和已有的重复。

> 文件丢进 `inbox/` → 对 Cursor Agent 说「把 inbox/xxx 按 SCHEMA 转成标准台本，更新 index，归档源文件」→ 双击 `start.bat` 预览 → `git add -A && git commit -m "..." && git push`。
